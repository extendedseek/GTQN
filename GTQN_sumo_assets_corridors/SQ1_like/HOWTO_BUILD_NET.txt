# SQ1_like SUMO scenario (corridor, 10 traffic lights)

This folder contains:
- SQ1_like.nod.xml (nodes)
- SQ1_like.edg.xml (edges)
- SQ1_like.rou.xml (time-varying flows similar to the paper schedule)
- sim.sumocfg
- tls_ids.json
- corridor.json

## Build the network (.net.xml) with netconvert (recommended by SUMO docs)
SUMO docs recommend generating `.net.xml` using netconvert/netgenerate, not manual editing.

Run (from this folder):

  netconvert --node-files SQ1_like.nod.xml --edge-files SQ1_like.edg.xml \
    --tls.guess --tls.guess.threshold 0 \
    --default.lanenumber 1 --default.speed 13.90 \
    --output-file SQ1_like.net.xml

Notes:
- `--tls.guess` creates traffic lights and a default program at nodes marked `type="traffic_light"`.
- You can replace the guessed TLS programs with your 4-phase program later using netedit or an additional tls file.


See SUMO Traffic Lights docs for how to load/override tlLogics.
